# PVU-1 — тестовое задание (Младший инженер IoT)

## Что сделал

1. Поднял ThingsBoard Community Edition через Docker Compose: два сервиса —
   PostgreSQL (с volume для хранения данных) и сама нода ThingsBoard, с
   `restart: unless-stopped`.
2. Создал в ThingsBoard тенанта, администратора тенанта и устройство `PVU-1`,
   получил его access token.
3. Написал на Node.js эмулятор приточно-вытяжной установки, который каждые
   5 секунд шлёт телеметрию в ThingsBoard по MQTT. Параметры не случайные —
   меняются во времени похоже на реальную работу установки: температура
   притока подтягивается к уставке через клапаны нагрева/охлаждения,
   вентиляторы плавно разгоняются и останавливаются, перепад давления на
   фильтре постепенно растёт (засорение) и изредка сбрасывается (замена
   фильтра), статус и авария зависят от состояния системы.
4. Собрал дашборд `PVU-1 Monitoring`: карточки текущих значений (температура
   притока, уставка, влажность, перепад давления на фильтре, статус, авария,
   засорение фильтра) и графики — температуры вместе, перепад давления на
   фильтре отдельно, обороты вентиляторов.
5. Проверил, что все параметры телеметрии доходят до ThingsBoard и дашборд
   обновляется в реальном времени.

## Настройки

- Образ: `thingsboard/tb-node:4.3.1.3`, БД: `postgres:18`
- Очередь: in-memory
- Интервал публикации телеметрии: 5000 мс
